NAME

FBB::Stat - Determines File Characteristics

SYNOPSIS

#include <bobcat/stat>
Linking option: -lbobcat

DESCRIPTION

FBB::Stat is a wrapper around the stat(2) system function. In particular, it offers features to test directly for file characteristics offered by stat(2).

NAMESPACE

FBB
All constructors, members, operators and manipulators, mentioned in this man-page, are defined in the namespace FBB.

INHERITS FROM

-

ENUMARATIONS

FBB::Stat::Combine:
This enumeration has the following values:

  • ALL: require all of the specified Mode or SpecialMode values to match;
  • ANY: require any match of the specified Mode or SpecialMode values (one match suffices);

FBB::Stat::Mode:
This enumeration has the following values:

  • UR: the owner of the file has read permissions
  • UW: the owner of the file has write permissions
  • UX: the owner of the file has execute permissions
  • GR: the group to which the file belongs has read permissions
  • GW: the group to which the file belongs has write permissions
  • GX: the group to which the file belongs has execute permissions
  • OR: others have read permissions
  • OW: others have write permissions
  • OX: others have execute permissions
  • RWX: all of the above.

FBB::Stat::SpecialMode:
This enumeration has the following values:

  • SUID: set UID bit is up
  • SGID: set GID bit is up
  • SB: sticky bit is up

FBB::Stat::Type:
This enumeration has the following values:

  • BLOCK_DEVICE: the object represents a block device
  • CHARACTER_DEVICE: the object represents a character device
  • DIRECTORY: the object represents a directory
  • FIFO: the object represents a named pipe (a queue)
  • REGULAR_FILE: the object represents a regular file
  • SOCKET: the object represents a socket
  • SYMBOLIC_LINK: the object represents a symbolic link

CONSTRUCTORS

  • Stat():
    The default constructor, creating an empty FBB::Stat object.
  • Stat(std::string const &fname):
    Initializes a FBB::Stat with a given file name.
  • Stat(std::string const &fname, std::string const &searchPath):
    Initializes a FBB::Stat with a given file name, where the file is searched in the searchPath directories, which is a colon-separated string of directory names. The filenames are constructed by appending fname to each of the elements of searchPath until an existing file is found. This file is then used. If fname is an absolute path, searchPath is ignored.
  • Stat(Stat const &other):
    The copy constructor is available.
