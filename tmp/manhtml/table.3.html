NAME

FBB::Table - Generates row- or column-wise filled tables

SYNOPSIS

#include <bobcat/table>
Linking option: -lbobcat

DESCRIPTION

FBB::Table objects may be used to create tables. The tables are filled either column-wise or row-wise. The class only has a minimal number of member functions and overloaded operators. Most of the table's characteristics may be find-tuned by a separate FBB::TableSupport object, described in a separate man-page (TableSupport(3bobcat)). When no FBB::TableSupport object is used, a plain row-wise or column-wise table will be inserted into a std::ostream.

NAMESPACE

FBB
All constructors, members, operators and manipulators, mentioned in this man-page, are defined in the namespace FBB.

INHERITS FROM

FBB::TableSpec - The class FBB::TableSpec is a mere support class for FBB::Table. Because of this, its features are repeated below.

ENUMERATIONS

The following enumerations are defined in the class FBB::TableSpec, allowing the programmer to specifiy values like FBB::TableSpec::Vertical rather than FBB::Table<std::istream_const_iterator>::Vertical.

enum FillDirection
This enumeration holds two values:

  • Horizontal:
    When this value is specified at construction time, subequent elements will be stored row-wise in the table. I.e., the second element will normally be in the second column of the first row.
  • Vertical:
    When this value is specified at construction time, subequent elements will be stored column-wise in the table. I.e., the second element will normally be in the second row of the first column.

enum WidthType
This enumeration holds two values:

  • EqualWidth:
    This value may be specified when all the table's columns should have equal width (i.e., equal to the width of the widest table element),
  • ColumnWidth:
    This value may be specified when the columns should be allowed variable widths. In this case each column will be as wide as its widest element.

CONSTRUCTORS

  • Table(Iterator const &begin, Iterator const &end, unsigned nColumns, TableSpec::FillDirection direction):
    This constructor expects two `Input Iterators' defining the range of elements to be displayed by the table. The elements to which the iterators point must be insertable as text into ostream objects. The parameter nColumns may be initialized with the number of columns the table should have. The number of rows are implied by the combination of this parameter and the number of elements in the iterator-range.
  • Table(Iterator const &begin, Iterator const &end, TableSupport &tableSupport, unsigned nColumns, TableSpec::FillDirection direction):
    This constructor operates identically to the previous constructor, but expects an additional reference to a FBB::TableSupport object. A FBB::TableSupport object offers additional formatting features used by the table to define elements like horizontal lines between rows, additional separators between elements etc.
The copy constructor is not available.

OVERLOADED OPERATORS

  • std::ostream &::operator<<(std::ostream &, FBB::Table<Iterator> const &table):
    This overloaded operator is actually an overloaded operator in the class FBB::TableSpec, but since FBB::TableSpec is the baseclass of FBB::Table, an FBB::Table<Iterator> object may be inserted into a std::ostream. Tables may be inserted repeatedly (e.g., using a sequence of insertions).

    Note that this operator is not defined in the namespace FBB.

The overloaded assignment operator is not available.

MEMBER FUNCTION

  • void setWidth(FBB::TableSpec::WidthType type):
    This member may be used to specify the width of indvidual or all columns of the table. Specify TableSpec::EqualWidth when all columns should have equal widths, specify TableSpec::ColumnWidth when each column should be as wide as its widest element. This member is actually a member of FBB::Table's baseclass FBB::TableSpec.

MANPULATORS

  • FBB::equalWidth(FBB::Table<Iterator> &table):
    This manipulator may be used to insert a table into a std::ostream object, enforcing equal column widths. To use this manipulator, the header file <bobcat/equalwidth> must be inserted.
  • FBB::columnWidth(FBB::Table<Iterator> &table):
    This manipulator may be used to insert a table into a std::ostream object, enforcing column widths which are as wide as their widest elements. To use this manipulator, the header file <bobcat/columnwidth> must be inserted.
